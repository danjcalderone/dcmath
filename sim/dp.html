Dynamic Programming Principle

Dynamic programming and related methods are the core of time domain optimization. The key idea is that since time only flows forward the best way to optimize actions over time is to figure out what is best to do at later times first then plan one's actions in the present considering both how much they will cost immediately and how well they will position one for actions in the future. The central optimization principle is represented using the famous Bellman equation. $$ V_{x,t} = \min_{u \in \mathcal{U}_{x,t}} \Big[\ell_{x,u,t} + V_{x^+,t+1} \ | \ x^+ = f(x,u,t) \Big] $$ Here, \(\mathcal{X}\) is some abstract state space and \(\mathcal{U}_x\) is some abstract set of actions that can be taken from any state \(x \in \mathcal{X}\). \(x^+\) is the next state we reach when we take action \(u\) \(\ell_{x,u,t}\) is an immediate cost that we incur when taking action \(u\) from state \(x\) at time \(t\) \( V_{x,t} \) is what we call a value function or cost-to-go at state \(x\) at time \(t\). This value function encodes how much we will pay from the current state at time \(t\) till the end of the time horizon if we do the optimal thing from now on. We notice that this equation is iterative in that the current value \(V_{x,t}\) depends on \(V_{x,t+1}\), the value at the next time step. This encodes the idea that we must optimize our current action based on both the current cost we will incur \(\ell_{x,u,t}\) and how that action positions us in the future. The final piece in this modeling framework is the dynamics of the system, ie. a description of how the system evolves given our actions Dynamics are often represented using some function $$ x^+ = f(x,u,t) $$ that tells us where the system will end up at the next time step \(t+1\) if we apply action \(u\) from state \(x\) at time \(t\).

The key idea of dynamic programming is that in order to computer \(V_{x,t}\) we need to first know \(V_{x^+,t+1}\). This leads to the optimality principle that we need to optimize starting at furthest time step that we can and working backwards in time.
